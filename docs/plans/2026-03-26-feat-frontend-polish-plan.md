# feat: Frontend Polish

## Overview

The app is functionally complete but visually minimal. Several quick-win UI improvements will make it feel finished: showing the AI-generated course title (currently computed but never displayed), structured lesson cards, a pill-button lesson selector, a progress bar during generation, and a branded page header. Two logic bugs discovered during planning are also addressed.

## Problem Statement

1. `courseMeta.courseTitle` is generated by Claude and saved to state, but **never displayed** in the UI.
2. The generating step shows raw log text with no overall progress indicator.
3. Lesson cards are unstyled textareas with a tiny gray metadata row — no visual structure.
4. The lesson count selector uses native HTML radio inputs that look out of place.
5. The page header is a plain `<h1>` with no visual anchor.
6. **Bug:** Partial-lesson recovery (stream catch block) restores lessons from sessionStorage but not `courseMeta`.
7. **Bug:** Progress bar denominator should come from the `selection` SSE event (`event.data.lessons.length`), not from `lessonCount` state — AI may select fewer lessons than requested.

## Proposed Solution

Six UI changes and two bug fixes, all within `src/components/features/ReviewForm.tsx` (plus one metadata line in `layout.tsx`). No new dependencies.

## Technical Approach

### Change 1 — Pill Buttons for Lesson Count Selector

Replace the `<input type="radio">` group (lines 296–310) with styled pill `<button>` elements.

```tsx
<div className="flex items-center gap-2 text-sm">
  <span className="font-medium text-gray-700">Length:</span>
  <div className="flex gap-1">
    {ALLOWED_LESSON_COUNTS.map(n => (
      <button
        key={n}
        type="button"
        onClick={() => setLessonCount(n)}
        className={[
          'rounded-full px-3 py-1 text-sm font-medium transition-colors',
          lessonCount === n
            ? 'bg-indigo-600 text-white'
            : 'bg-white border border-gray-300 text-gray-600 hover:border-indigo-400 hover:text-indigo-600',
        ].join(' ')}
      >
        {n}
      </button>
    ))}
  </div>
  <span className="text-gray-400">lessons</span>
</div>
```

Note: `type="button"` is critical — prevents accidental form submission.

### Change 3 — Generating Step: Progress Counter + Bar + Bug Fixes

Add `expectedLessonCount` state (set on `selection` SSE event) to track the real denominator:

```tsx
const [expectedLessonCount, setExpectedLessonCount] = useState<number>(lessonCount)
```

On the `selection` event handler, add:
```tsx
setExpectedLessonCount(event.data.lessons.length)
```

Derive `completedCount` from `streamLog`, filtering on `✓ Lesson ` (not just `✓`) to avoid false matches from course titles containing a checkmark:

```tsx
const completedCount = streamLog.filter(l => l.startsWith('✓ Lesson ')).length
```

Replace the generating step JSX (lines 321–335):

```tsx
{step === 'generating' && (() => {
  const completedCount = streamLog.filter(l => l.startsWith('✓ Lesson ')).length
  return (
    <div className="space-y-4">
      <div className="flex items-center justify-between">
        <p className="text-sm font-medium text-gray-700 animate-pulse">Generating your course…</p>
        {completedCount > 0 && (
          <span className="text-sm font-semibold tabular-nums text-indigo-600">
            {completedCount} / {expectedLessonCount} lessons
          </span>
        )}
      </div>
      {completedCount > 0 && (
        <div className="h-1.5 w-full rounded-full bg-gray-100">
          <div
            className="h-1.5 rounded-full bg-indigo-500 transition-all duration-500"
            style={{ width: `${(completedCount / expectedLessonCount) * 100}%` }}
          />
        </div>
      )}
      {slowWarning && (
        <div className="rounded-md bg-amber-50 border border-amber-200 px-4 py-3 text-sm text-amber-700">
          This is taking longer than usual — still working…
        </div>
      )}
      <ul className="space-y-1.5 text-sm">
        {streamLog.map((line, i) => (
          <li key={i} className={['flex items-start gap-2', line.startsWith('✓') ? 'text-gray-700' : 'text-gray-400'].join(' ')}>
            {line.startsWith('✓')
              ? <span className="mt-0.5 text-green-500 shrink-0">✓</span>
              : <span className="mt-0.5 shrink-0 text-gray-300">·</span>
            }
            <span>{line.startsWith('✓') ? line.slice(2) : line}</span>
          </li>
        ))}
      </ul>
    </div>
  )
})()}
```

Note: Progress bar uses `style={{ width: ... }}` — Tailwind v4 doesn't support arbitrary dynamic fractions via JIT.

`expectedLessonCount` must also be reset in two places:

- Top of `handleGenerate` (before fetch, as the initial value):
  ```tsx
  setExpectedLessonCount(lessonCount)
  ```
- Inside `handleStartOver`:
  ```tsx
  setExpectedLessonCount(lessonCount)
  ```

### Change 5 — Lesson Cards with Visual Structure

Replace the lesson map (lines 351–366):

```tsx
{lessons.map((lesson, i) => (
  <div key={lesson.lessonNumber} className="rounded-xl border border-gray-200 bg-white shadow-sm overflow-hidden">
    <div className="flex items-center gap-3 px-4 py-3 border-b border-gray-100 bg-gray-50">
      <span className="flex h-6 w-6 shrink-0 items-center justify-center rounded-full bg-indigo-600 text-xs font-bold text-white">
        {lesson.lessonNumber}
      </span>
      <div className="flex-1 min-w-0">
        <p className="truncate text-sm font-medium text-gray-800" title={lesson.title}>{lesson.title}</p>
        <p className="truncate text-xs text-gray-400 mt-0.5">Subject: {lesson.subjectLine}</p>
      </div>
      <span className="shrink-0 font-mono text-xs text-gray-300 hidden sm:block">{lesson.filename}</span>
    </div>
    <textarea
      value={lesson.markdownBody}
      onChange={e => handleLessonEdit(i, e.target.value)}
      rows={20}
      className="w-full px-4 py-3 font-mono text-xs focus:outline-none focus:ring-2 focus:ring-inset focus:ring-indigo-500 resize-y"
    />
  </div>
))}
```

Key details:
- `title={lesson.title}` on the truncated `<p>` provides tooltip for long titles.
- Filename hidden on mobile (`hidden sm:block`) to prevent layout overflow on narrow viewports.
- `focus:ring-inset` prevents the textarea focus ring from clipping against the card border.
- Cards use `bg-white shadow-sm` against the `bg-gray-50` page — natural depth without heavy shadows.
